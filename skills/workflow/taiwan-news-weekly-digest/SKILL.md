## Overview

本 skill 針對台灣產業新聞進行週度彙整，特別適合需要快速把握市場脈動的決策者與分析師。

相比單日分類（由 `taiwan-news-classifier` 負責），本 skill 輸出的是**週度視角**：
- 每日主題提取：標示該天的核心議題（如「半導體漲價」「AI 投資加速」）
- 高頻標題識別：找出跨日期重複出現或業界廣泛討論的新聞
- 結構化周報：在一張表格內呈現 5-7 天的完整輪廓

## 何時使用

### 典型觸發場景

1. **週一早會準備**：「上週台灣科技產業有哪些重點新聞？」
   - 主管要在會議上提及市場背景
   - 需要 30 秒快速掃過上週熱點

2. **投資決策前置**：「最近一週與 AI、電池相關的新聞高峰？」
   - 準備簡報時確認產業熱度
   - 確認市場是否有新的風險信號

3. **行業分析彙整**：「過去 7 天各產業的重點標題分別是什麼？」
   - 製作行業週報時的素材蒐集
   - 對標競爭對手的公開討論角度

4. **新聞追蹤驗證**：「這個星期有沒有關於 X 產業的高頻新聞？」
   - 確認某個事件是否持續發酵
   - 判斷是孤立事件還是系統性趨勢

## 執行步驟或模式

### 基本工作流

**第 1 步：定義時間範圍**
- 預設過去 7 天（今日往前推 168 小時）
- 可視情調整為 5 天或 10 天
- 明確起訖日期（如 2026-06-06 ~ 2026-06-13）

**第 2 步：蒐集新聞清單**
- 從台灣主要產業新聞源（經濟日報、工商時報、新華社台灣版、科技新報等）
- 篩選與「上市/上櫃公司、產業重組、政策變動」直接相關的新聞
- 過濾掉純評論或重複報導（同一則新聞多家轉載只算一則）

**第 3 步：按日期分組**
- 新聞按刊登日期（非發佈日期）排序
- 同日多則新聞視為「該日新聞集合」
- 建立日期 → 新聞標題的映射表

**第 4 步：提取每日主題**
- 檢視該日新聞集合，找出 2-4 個共同議題
- 議題名稱需簡明且具體（如「半導體漲價」而非「價格波動」）
- 若當日新聞量少（< 3 則）或過於離散，標記為「無明顯主題」

**第 5 步：識別高頻標題**
- 統計全週標題中重複出現的主題（同一企業、同一事件多日報導）
- 跨日期追蹤：「X 公司」出現 3+ 則、「Y 政策」出現 2+ 日的視為「高頻」
- 按照頻率與時間新鮮度排序

**第 6 步：組成週度摘要表**

格式範例：

```
週度產業新聞摘要（2026-06-06 ~ 2026-06-13）

| 日期 | 主題 | 高頻標題 |
|------|------|---------|
| 2026-06-06（五）| 法令遵循、電池供應鏈 | 台積電宣告新廠計劃、國際電池標準將生效 |
| 2026-06-07（六）| 無明顯主題 | （新聞量少，分類待補） |
| 2026-06-08（日）| AI 投資升溫、晶片需求 | OpenAI 釋出新模型、客戶詢價激增 |
| 2026-06-09（一）| 匯率衝擊、成本壓力 | 新台幣升值對出口影響、原物料價格上升 |
| 2026-06-10（二）| 政策支持、綠能轉型 | 政府公告再生能源補助延長、廠商併購案 |
| 2026-06-11（三）| 人才爭奪、薪酬調整 | 科技廠陸續加薪 5-8%、獵人頭公司業績增 |
| 2026-06-12（四）| M&A 活躍、產業整併 | 三家中型上市櫃進行策略談判、整併預告 |

高頻標題排行（跨日追蹤）：
1. AI 晶片需求 → 5 則新聞、4 個日期
2. 台積電動向 → 3 則新聞、2 個日期
3. 政府政策支持 → 3 則新聞、2 個日期
```

**第 7 步：產出最終報告**
- 以表格形式呈現上述內容
- 高頻標題按熱度排序並標註出現次數
- 備註說明（如「受假期影響新聞量較少」）
- 提供原始新聞連結清單供進一步查閱

### 與 taiwan-news-classifier 的協作

- **Taiwan-news-classifier** 的產出（單日分類 + 子類別）作為本 skill 的輸入源
- 本 skill 在 classifier 的「相關新聞」清單基礎上，進行**時間序列聚合與主題提升**
- 若無 classifier 前置結果，可直接蒐集新聞後進行週度彙整（成本較高）

## 注意事項

### 資料品質與時效性
- **新聞源延遲**：台灣主流媒體通常於工作日刊登，週末假日新聞量較少。若跨假期進行週度統計，需標註「假期影響」
- **重複報導**：同一則新聞被多家媒體轉載時，僅在高頻標題中計算一次，避免數字虛高
- **破名言限制**：部分新聞付費牆後可能無法即時抓取，建議預留人工複查時間

### 主題提取的主觀性
- 「每日主題」提取涉及主觀判斷，建議在關鍵決策前人工驗證
- 若當日新聞涉及多個不相關產業（如「房產政策」與「航運股漲停」同日），應分別列出而非強行合併

### 高頻標題的閾值設定
- 預設「高頻」為 2+ 日期或 3+ 則新聞，可根據新聞總量調整
- 新聞量少週（< 15 則）時，可降低閾值至 2 則新聞；新聞量多週（> 50 則）時，提升至 4 則或更多
- 明確記錄本週所用的閾值供追蹤

### 與投資決策的距離
- 週度摘要呈現的是「市場討論」而非「市場現實」
- 高頻標題反映媒體關注度，未必代表實際發生或影響大小
- 用於初步篩選與背景確認，不應直接作為交易或配置決策的依據

### 追蹤一致性
- 每週固定時間產出（如每週一早上 8 時）有助於比較趨勢
- 若變更新聞源、主題定義或高頻閾值，應在報告中標註，確保歷史可比性
