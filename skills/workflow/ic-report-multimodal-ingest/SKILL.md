## Overview

本 skill 解決投資評估流程中的「多模態輸入 normalize」問題。當評估報告同時存在正式 PDF 版本與會議現場補充照片時，手動整理耗時且容易遺漏。本 skill 一次讀完所有來源，自動提取關鍵資訊，輸出統一的結構化 IC memo 草稿（包括標的、產業、財務摘要、風險、決議建議）。

相比既有的 `tw-company-pe-memo` 與 `tw-company-pe-memo-refine`（需要已知公司再產出），本 skill 先對多份不同格式的輸入做正規化與信息融合，然後可選搭配上述 skills 深化分析。

## 何時使用

### 具體觸發場景

1. **IC 會議結束，要快速整理決議材料**  
   投資委員會上有人帶了印刷版報告，會議中用手機補充拍照新發現（如附註頁、競爭對手數據），散開後需要統一電子化版本。

2. **PDF 報告來自外部顧問，JPG 是公司內部補充意見**  
   例如第三方投資顧問提供正式 PDF，但公司融資負責人現場用手機拍攝補充資料頁面（月營收圖表、客戶清單等），要合併成單一評估文檔。

3. **同一份標的，多場評估會議的累積材料**  
   標的經歷複輪融資，每輪都有正式報告 PDF，加上不同時點的現場補充照片，要產出統一的 timeline 化 IC memo。

### 觸發訊號

- 使用者在工作目錄貼上 **1 個 `*.pdf`** 檔案
- 同時貼上 **1 或多個 `*.jpg` / `*.jpeg` / `*.png`** 檔案，且**至少一個檔名符合 `YYYYMMDD_HHMMSS` 或 `YYYYMMDD_*` 格式**（表示時間戳）
- 使用者明確說「整理成 IC 評估」、「產生 IC memo」、「彙整這份報告」、「合併這些文件」等要求

## 執行步驟與模式

### 步驟 1：驗證與準備輸入

當觸發條件符合時，執行以下檢查：

```
1. 確認工作目錄中確實有 1 個 PDF（不超過 1 個）
2. 確認有至少 1 個 JPG/PNG（無上限）
3. 記錄各檔案的建檔時間或修改時間，作為讀取順序參考
   （通常 PDF 是主文檔，JPG 按時間戳遞增讀取）
4. 若 JPG 檔名含 YYYYMMDD_HHMMSS，按時間戳排序；若無，按字母順序
```

### 步驟 2：多模態讀取與提取

使用以下流程並行讀取所有文件：

**2a. PDF 內容提取**
- 調用 `/pdf` skill 或 `office-pdf` skill 完整讀取 PDF
- 輸出：`[標題, 目錄, 章節標題, 正文段落, 表格, 圖表標題]`
- 若 PDF 有表格（財務、市場規模等），優先標記為「表格內容」供後續結構化

**2b. JPG/PNG 光學字元識別（OCR）與視覺理解**
- 對每張照片調用 vision 或 OCR 工具（推薦 Claude Opus 或 Sonnet 的視覺能力）
- 提取：手寫註記、圖表截圖中的數字、補充表格、簽名/時間戳
- 記錄各照片的推斷時間點（若檔名含時間戳則直接使用）

**2c. 內容融合順序**
```
PDF（主骨架）
  ↓
JPG #1（時間最早的補充）→ 融合進對應章節
  ↓
JPG #2（時間次早）→ 融合
  ↓
...JPG #N
```

### 步驟 3：結構化 IC Memo 輸出

將融合後的內容組織成以下統一結構（markdown 格式）：

```
# IC 評估 Memo

## 一、標的基本資訊
- **公司名稱**：[從 PDF 和 JPG 中綜合判斷]
- **產業分類**：[行業、子細分]
- **商業模式簡述**：[1-2 句核心定位]
- **估值階段**：[融資輪次、當前市值等]
- **上報日期**：[YYYY-MM-DD，以最晚的 JPG 時間戳或 PDF 內容為準]

## 二、產業與市場
- **市場規模**：[全球/區域市場規模及成長率]
- **競爭格局**：[主要玩家、差異化點]
- **產業趨勢**：[關鍵驅動因子、威脅因素]

## 三、財務摘要
- **收入**：[過去 12M / 最近完整年度]
- **毛利率 / 淨利率**：[百分比]
- **現金流狀況**：[正負、燒錢率 if 虧損]
- **融資需求與用途**：[本輪融資額、資金用途]

## 四、核心風險
- **產品/技術風險**：[技術可行性、市場接受度]
- **團隊風險**：[創辦人背景、關鍵人才缺口]
- **財務風險**：[現金跑道、單一客戶依賴]
- **市場與監管風險**：[產業監管變化、市場競爭激化]

## 五、IC 決議建議
- **推薦級別**：[高度推薦 / 推薦 / 中立 / 不推薦]
- **關鍵決策因素**：[3-5 條影響投票的核心理由]
- **後續 Due Diligence 優先項**：[若決議推進，需進一步驗證的事項]
- **後續追問**：[給管理層或投資顧問的開放式提問]

## 六、來源與追溯
- PDF 來源：[檔名、頁數範圍]
- JPG 補充：[各檔名及其提供的具體資訊（如「第 5 張照片提供了 2025 年月營收圖表」）]
- 讀取時間：[YYYY-MM-DD HH:MM:SS，當前時間]
```

### 步驟 4：品質檢查與補充

在輸出前，執行以下驗證：

1. **覆蓋度檢查**：五個主要章節是否都有內容？若某章節無關鍵資訊，標記為 `[待補充]`
2. **重複檢查**：若 PDF 和 JPG 重複提及同一信息，保留最新版本，註記版本差異
3. **數字一致性**：若財務數據在 PDF 和 JPG 中矛盾，以最晚時間戳為準，並標註差異
4. **可追溯性**：每條關鍵事實後以 `[source: PDF p.X]` 或 `[source: JPG #N]` 註記

### 步驟 5：可選後續流程

若完成上述結構化後，可根據需要：

1. **搭配 `tw-company-pe-memo-refine` 深化分析**  
   將輸出的 IC memo 作為輸入，對標的進行財務模型驗證、市場假設檢驗。

2. **生成決議投票模板**  
   將「決議建議」自動轉換成 IC 投票時的選項與計分表。

3. **存檔並同步到共享文檔**  
   將 memo 存為 `ic_memo_[公司名]_[日期].md`，並同步到 Notion / Google Drive（可選）。

## 注意事項與已知限制

### 運作限制

1. **PDF 可讀性**  
   若 PDF 是掃描件（無法提取文本），需依賴 OCR。當前 OCR 對手寫筆記或複雜表格的準確率有限（<90%），建議人工驗證數字部分。

2. **JPG 時間戳依賴**  
   若 JPG 檔名不含時間戳（YYYYMMDD_HHMMSS），系統將按字母順序處理。建議使用者在貼上照片前檢查檔名，或在對話中標明各圖的來源順序。

3. **多語言混合**  
   若 PDF 是英文、JPG 上有中文手寫註記（反之亦然），OCR 輸出可能包含混合結果。最終 memo 優先用正體中文輸出，但會保留原語言的關鍵術語。

4. **複雜表格與圖表**  
   超過 10 行 × 10 列的表格可能無法完美提取；複雜財務模型（含多層公式）在 JPG 中難以準確識別。建議人工複查。

5. **機密資訊**  
   本流程會讀取並整理所有輸入文件內容。若報告含簽署協議的機密條款，請自行脫敏後再輸入。

### 工作流程提示

- **檔案大小**：單個 PDF ≤ 50MB、單張 JPG ≤ 10MB 為佳；超大文件可能導致提取耗時延長（>5 分鐘）。
- **等待時間**：3-5 張 JPG + 30 頁 PDF 的完整流程通常需要 2-3 分鐘。
- **輸出格式**：memo 優先輸出 markdown，便於後續編輯和版本控制。

### 與既有 Skills 的協作

- **`tw-company-pe-memo` 差異**：該 skill 需要已知公司名（API 查詢），本 skill 從多份文件綜合推斷。兩者可串聯：本 skill 先產出草稿，`tw-company-pe-memo` 再精化。
- **`office-pdf` 差異**：`office-pdf` 用於單純 PDF 讀取，本 skill 增加 JPG 融合與結構化 IC memo 輸出。
- **`5-local-first-pdf-excel-company-extractor` 差異**：該 skill 重點在公司名冊提取，本 skill 重點在評估內容融合與結構化。
