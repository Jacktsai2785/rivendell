## Overview

tw-company-name-extractor 是一個輕量級文本處理 skill，用來從任意非結構化文字中自動抽取台灣公司及機構名稱，輸出結構化 JSON 陣列。

相比於 `tw-company-lookup`（需已知公司名稱才能查詢）或 `tw-company-identify`（單一名稱驗證），本 skill 的價值在於**盲目發現**：給定一份文件，自動挑出其中涉及的所有台灣公司，無需事先知道有哪些公司。適用於：

- **DD 流程前置**：在進行詳細盡職調查前，先自動掃出新聞稿、法說、新聞中提及的所有相關企業，加速目標識別與範圍確認
- **投資組合監控**：定期掃描新聞內容，自動發現與現有投資組合相關的公司互動、競爭或並購活動
- **合約分析**：快速識別合約中的所有參與方（甲乙丙丁 之外還有多少方實際涉及）
- **市場情報**：從產業新聞、資助公告、政府補助清單中批量提取關鍵企業

純 LLM 推理、無外部工具依賴，執行速度快、成本低。

## 何時使用

### 典型觸發場景

1. **批次新聞掃描**  
   你有 50 篇與某個產業或議題相關的新聞標題和摘要，需要一次性列出其中提及的所有台灣上市櫃、準上市或知名企業。輸出一份清單供進一步分析。

2. **DD 前期快速掃描**  
   投委會通過一樁新的投資案件、獲得初步計劃書或新聞稿，需要快速識別其中涉及的所有相關企業（投資方、競爭者、供應商等），作為後續深度調查的起點。

3. **合約或公告分析**  
   政府補助公告、聯合新聞稿、或內部會議紀錄提到「與 A 公司、B 機構合作」等，需要把所有提及的機構名稱自動列出，確保沒有遺漏潛在的利益相關者。

4. **投資組合監控告警**  
   設定定期掃描（例如週報新聞）的流程，自動抓出其中涉及你投資組合企業的新聞內容，進一步與 `tw-company-lookup` 交叉驗證。

### 不適合使用的場景

- 只需驗證「某個特定名稱」是否為有效台灣公司 → 用 `tw-company-identify`
- 已知公司名稱，需要查詢註冊地、統編、董事會等詳細資訊 → 用 `tw-company-lookup`  
- 需要 100% 準確驗證，不容許任何誤識別 → 應結合人工審核或 `tw-company-identify` 驗證後的結果

## 執行步驟與模式

### 基本用法：單次文字分析

```
輸入文本範例：
「聯發科與台積電今日宣布深化合作，共同開發 5G 晶片。鴻海精密受惠於此，股價上漲 3%。
分析師認為，高通、博通等國際大廠也將跟進。聯詠科技則表示將聚焦顯示驅動晶片。」

預期輸出（JSON）：
[
  { "name": "聯發科", "confidence": 0.95, "type": "company" },
  { "name": "台積電", "confidence": 0.95, "type": "company" },
  { "name": "鴻海精密", "confidence": 0.93, "type": "company" },
  { "name": "聯詠科技", "confidence": 0.90, "type": "company" }
]
```

### 批次處理模式

如需處理多份文件（例如 30 份新聞），可逐件調用本 skill，最後匯聚結果：

1. 逐份文件調用 tw-company-name-extractor，收集各份的公司清單
2. 去重（同一個公司可能在多份新聞中出現）
3. 按 confidence 排序或按出現頻率統計
4. 可選：調用 `tw-company-identify` 或 `tw-company-lookup` 驗證高優先度的公司

### 與其他 skills 的組合

**提取 + 驗證流程**（確保高可信度結果）：
1. 用 tw-company-name-extractor 從文本中提取 candidates（confidence >= 0.8）
2. 用 `tw-company-identify` 驗證 candidates（是否為實際登記公司）
3. 用 `tw-company-lookup` 查詢確認者的詳細信息（統編、董監事等）

**提取 + 監控流程**（投資組合追蹤）：
1. 用 tw-company-name-extractor 掃描新聞/公告
2. 將結果與投資組合清單交叉比對，標記涉及投資組合公司的新聞
3. 推送告警或彙整為週報

## 注意事項

### 已知限制

1. **誤識別風險**  
   - 可能將某些人名（例如「馬英九」）誤認為公司（馬英資訊、馬英科技等）
   - 地名或常見詞組偶爾被誤判（例如「東方」、「富邦」既是地名也是公司名）
   - 解決方式：對 confidence 低於 0.80 的結果應人工審核，或進一步用 `tw-company-identify` 驗證

2. **舊名稱或非官方名稱**  
   - 只能識別常見的公司名稱，對舊名稱、暱稱、俗稱的識別度可能較低
   - 若文本使用「聯電」而非「聯華電子」，可能無法識別或 confidence 會降低
   - 解決方式：若業務需要涵蓋別稱，應事先在提示中補充別稱對應表

3. **文本編碼與破損**  
   - 如果輸入文本包含異常的繁簡混用、亂碼或特殊符號，提取結果可能失準
   - 建議預先進行文本清理（移除不可見字符、統一編碼）

4. **沒有驗證能力**  
   - 本 skill 純粹基於名稱識別，**不驗證公司是否真實存在**、是否為上市櫃企業、是否仍在營運
   - 例如可能識別到一家已下櫃或已解散的公司
   - 若需驗證，必須搭配 `tw-company-identify` 或 `tw-company-lookup`

5. **小規模或新成立企業識別度低**  
   - 訓練資料偏向知名大型企業，對於新成立公司、小型企業、微型企業的識別度有限
   - 如需完整涵蓋所有企業規模，應結合其他識別方式（例如 KIPO 公開資料比對）

### 使用最佳實踐

- **設定適當的 confidence 閾值**：生產流程建議只採納 confidence >= 0.85 的結果，0.80–0.85 之間的應進行人工審核
- **去重與標準化**：批量結果中可能出現「聯發科技」與「聯發科」，應事先定義標準化規則或使用 `tw-company-identify` 統一名稱
- **定期驗證**：若用於持續監控（如週報），每月抽樣驗證 10% 的結果，確保識別質量未下滑
- **補充上下文**：如果原文已包含公司統編或其他識別碼，應連同名稱一併傳入 skill，提高精準度
