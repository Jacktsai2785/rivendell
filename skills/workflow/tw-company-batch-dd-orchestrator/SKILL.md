## Overview

批次 DD orchestrator 自動化連接兩個核心工具，將原本「手動調度 finder session → 蒐集結果 → 手動調度 PE memo session」的流程整合為一條流：

1. 接收公司名單（JSON、CSV 或結構化文本）
2. 為每家公司依序執行 **tw-company-website-finder**（查官網、營運狀況、市場定位）
3. 使用 finder 結果作為 **tw-company-pe-memo** 的輸入上文
4. 蒐集所有 PE 備忘錄並彙整成單一報告

相比逐家手動調度，批次 orchestrator 省掉報告蒐集、結果合併、進度追蹤的重複工作，尤其適合 10+ 公司規模的盡調清單。

## 何時使用

**適用場景：**
- 基金正在評估某個賽道的多家潛在被投資對象（品牌電商、SaaS、新能源等）
- 手上有一份未經篩選的候選清單（來自招商資料、業界推薦、前期掃描）
- 需要快速為每家公司生成 PE 視角的備忘錄，供投委會或合夥人決策參考
- 時間框架在 1–2 週內完成分析（避免資訊老化）

**不適用場景：**
- 只評估 1–2 家公司（直接逐家手動執行更靈活）
- 清單中公司數 > 100（單日 API 額度可能不足，建議分週期）
- 公司全為非國內企業或無官網（finder 會返回空結果，浪費配額）

**準備工作：**
1. 整理公司名單，確保公司名稱完整且無重複（去重後再執行）
2. 檢查清單長度：10–50 家的批次最優（完成時間 2–4 小時，API 配額控制良好）
3. 確認所有公司都有官網或可公開查詢的基本資訊（否則 finder 返回「未找到」，PE memo 會基於空上文生成）

## 執行步驟或模式

### 步驟 1：準備公司名單

將公司名單以以下任一格式提供：

**格式 A：JSON 陣列**
```json
{
  "companies": [
    {"name": "公司 A", "description": "市場定位或評估背景（選填）"},
    {"name": "公司 B"},
    {"name": "公司 C"}
  ]
}
```

**格式 B：CSV（公司名稱為第一欄）**
```
公司名稱,背景或備註
公司 A,潛在並購標的
公司 B,競品分析
```

**格式 C：純文本列表**
```
1. 公司 A
2. 公司 B
3. 公司 C
```

### 步驟 2：啟動 Orchestrator

調用 skill：
```
/tw-company-batch-dd-orchestrator
```

系統提示輸入：
- 公司名單（可直接貼上 JSON、CSV 或列表）
- 評估背景或專案代碼（選填，用於報告追蹤）
- 預計報告儲存位置（預設為 `./reports/batch-dd-{timestamp}.md`）

### 步驟 3：自動流程執行

**Phase 1：網站查找**
- 逐家公司調用 `tw-company-website-finder`
- 蒐集官網、營運資訊、市場定位、聯絡方式
- 失敗重試最多 2 次；若仍失敗，記錄為「未找到」並繼續下家

**Phase 2：PE 盡調備忘錄**
- 將 Phase 1 結果作為上文上下文
- 調用 `tw-company-pe-memo` 生成投資視角分析（市場、競爭、風險、建議）
- 每份備忘錄獨立存檔，同時累積到彙總報告

**Phase 3：報告彙整**
- 匯總所有 PE 備忘錄
- 生成執行摘要（評估批次規模、成功率、主要風險集群）
- 內含按公司索引表，便於搜尋和決策參考

### 步驟 4：成果物檢查

報告儲存位置：`./reports/batch-dd-{YYYY-MM-DD-HHmmss}.md`

報告結構：
```
# 批次 PE 盡調報告
- 評估範圍：10 家公司
- 執行時間：2026-06-13 10:30–14:45
- 成功率：8/10（2 家官網查找失敗）
- API 配額消耗：17/50（finder + memo）

## 執行摘要
- [風險總結]
- [賽道觀察]
- [推薦決策]

## 公司索引
| 排序 | 公司 | 官網 | PE 評分 | 主要風險 |

## 詳細分析
### 1. 公司 A
[tw-company-website-finder 結果摘要]
[tw-company-pe-memo 完整備忘錄]

### 2. 公司 B
...
```

## 注意事項

### API 配額與速率限制
- 單日 tw-company-website-finder 限額：50 次
- 單日 tw-company-pe-memo 限額：50 次
- 50 家公司需耗 50 + 50 = 100 次配額，超出單日限制
- **建議分批執行：**第一週 30–40 家，第二週補足剩餘，避免配額透支

### 網站查找失敗的處理
- 若 finder 返回「官網未找到」，orchestrator 將標記為 ⚠️，PE memo 將基於「市場推測」生成備忘錄
- 建議在報告中手動補充已知資訊（年營收、股東背景、近期融資等），提升分析品質
- 失敗不中斷流程，會繼續處理後續公司

### 中斷與重啟
- 若執行途中中斷（網路斷線、配額耗盡），orchestrator 記錄進度狀態文件 `.batch-dd-checkpoint.json`
- 重新執行時系統自動檢查 checkpoint，跳過已完成的公司，僅繼續未完成部分
- Checkpoint 檔案儲存在執行目錄下，刪除後重新執行會重頭開始

### 報告合併與去重
- 若兩批次無意間評估了相同公司，orchestrator 在最終報告中合併結果（保留最新 memo，舊版本存檔為 `_deprecated_`)
- 同批次內若有重複公司名稱（如「騰訊」vs「Tencent」），系統視為不同記錄；建議事前手動去重

### 性能與時間估計
- 平均單家公司執行時間：15–25 分鐘（finder 5–10 分鐘 + memo 10–15 分鐘 + 併發延遲）
- 10 家：2.5–4 小時
- 20 家：5–8 小時
- 建議晚間或週末執行，避免高峰期 API 延遲

### 與手動流程的兼容性
- Orchestrator 產生的中間結果（finder 輸出、PE memo）都可手動編輯再上傳回最終報告
- 若需調整某家公司的 memo，可選擇「跳過此家、稍後手動補充」，不中斷整批流程
