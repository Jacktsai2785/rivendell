**觸發語：** 「孤兒 skill」、「盤點所有 skill」、「清理無用 skill」、「orphaned skills」、「workflow-map 不同步」

**分類：** `meta`

3. 交叉比對 → 分類：(a) 有檔案無登錄、(b) 有登錄無檔案、(c) 兩者吻合

### 2. `skill-completeness-validator` ⭐ Moderate

**目的：** 掃描所有 SKILL.md 前言（frontmatter），確認必填欄位（name、description、觸發語、DO NOT TRIGGER 段落）均已填寫且內容不為空，避免 harvest / server 因解析不到描述而報錯。

**觸發語：** 「skill 描述不完整」、「validate skill metadata」、「skill 品質檢查」、「SKILL.md 漏欄位」

**分類：** `quality`

**理由：** Session 3（27 則訊息）起因是「完全沒有詳細說明的奇怪 skill」導致 server.py 錯誤，最終需要手動修復 harvest-2026-06-13.md 和 server.py。這個場景可預防：在 skill 寫入後立即跑驗證，而非等到 harvest/server 炸掉才發現。  
核心邏輯：YAML frontmatter 解析 → 必填欄位清單比對 → description 長度/觸發語有無 → 輸出問題清單 → 可選自動補齊樣板。

**與現有 skill 的差異：**
- `skill-creator` → 從頭建立 skill，不做已存在 skill 的驗證
- `audit-fix` → 修 permissions，不讀 SKILL.md 內容
- `de-slopify` → 改善文字品質，不做結構驗證

**重複風險：** 低
