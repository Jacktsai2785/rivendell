### ✅ Strong：`repo-rename`

**目的**：將一個 git repo 改名時，系統性地審計所有跨位置的引用並產生遷移清單

**觸發條件**：使用者說「把 repo 改名為 X」、「repo 改名」、「rename project」、`/repo-rename`

**分類**：`meta` / git 工具

**來源 session**：`subagents/agent-a08573db`、`skills-test/1350a5c9`（討論改名 rivendell）

**理由**：
session 中出現了完整的 repo 改名審計流程，且需要同時掃描多個不同類型的位置：
1. `~/Library/LaunchAgents/*.plist` — agent 排程設定
2. `~/.claude/projects.json` — Claude Code 專案索引
3. `bin/` / `scripts/` — shell 腳本內嵌路徑
4. `.claude/settings.local.json` — 本地設定
5. 其他專案的 grep（跨 repo 引用）
6. `package.json`、`pyproject.toml`、`next.config.ts` 等設定檔
7. `git remote -v` — GitHub remote URL

這是一個明確的多步驟、多工具序列，且每次改名都會重複執行。目前沒有任何 skill 涵蓋此流程。不同於 `init-project`（初始化），`repo-rename` 是「重構現有 repo 身分」的全域審計工作。

**Skill 核心工作流**：
```
1. 掃描 ~/Library/LaunchAgents/*.plist → grep {old_name}
2. 掃描 ~/.claude/projects.json → 找 session key
3. 掃描 bin/ scripts/ .claude/ → 找 hardcoded path
4. 掃描 package.json / pyproject.toml / settings files
5. 掃描兄弟 repos → grep {old_name}
6. 檢查 git remote URL
7. 輸出：遷移清單（每個引用 + 修改方式）
8. 提示：launchctl unload → rename dir → launchctl load 的操作順序
```

### 🟡 Moderate：`knowledge-inbox`

**目的**：為知識庫管理導入 inbox/library 兩階段工作流（inbox 暫存 → review → promote 到 library）

**觸發條件**：使用者說「設計知識管理」、「加 inbox 流程」、「知識分類工作流」、`/knowledge-inbox`

**分類**：`backend` / 資料管理

**來源 session**：`sales-assistant/81b9ce5c`

**理由**：
session 中完整實作了一套 inbox/library 模式：
- 所有新進知識預設寫入 `knowledge/inbox/`（未分類暫存區）
- 使用者 review 後，透過 `POST /api/nx/memory/promote` 升格到 `knowledge/library/`
- 每個 `.md` 檔使用 YAML frontmatter 標記 `status: inbox/library`
- 後端提供 `/categories`、`/promote`、`/demote` API
- 前端頁面有 inbox/library 分頁切換 + 搜尋過濾

這個模式解決了知識檔案累積時「品質不均」的問題，適用於任何有知識庫的 AI 輔助專案。與現有 `knowledge-graph` skill（entity tracking JSONL 格式）定位不同，此 skill 管理的是 **人工審閱的 markdown 知識文件**。

**與現有 skill 的差異**：
- `knowledge-graph`：JSONL entities（自動萃取），無人工 review 流程
- `knowledge-inbox`：Markdown 文件（人工輸入），有 staging → promote 工作流

**候選實作範圍**：
- inbox/library 目錄結構設計
- YAML frontmatter schema（`status`, `category`, `created`, `tags`）
- Promote/demote API endpoint pattern
- 前端分頁元件 blueprint
