## Skill 候選清單

### 候選 1：`3-jk-nb-lint`　｜ 評級：**Moderate**

| 欄位 | 內容 |
|------|------|
| **目的** | 執行 jk-nb wiki 的 `lint` 動詞——掃描整個 wiki/ 目錄，找出內容品質問題並輸出可執行的審計報告 |
| **觸發條件** | 使用者說「跑 lint」「wiki lint」「知識庫品質檢查」「jk-nb lint」「找矛盾」「清 dead link」，或 working directory 是 jk-nb 且用戶提到品質問題 |
| **分類** | workflow（與 `1-llm-wiki-knowledge-base`、`2-jk-nb-consume` 同一技能序列） |
| **輸出** | `_lint-report.md`（7 類問題清單）、`agent-actions.md`（append 待辦行動） |

**檢查項目（來自 Session 1）：**
1. 互相矛盾的敘述（跨頁比對）
2. 跨頁重複內容
3. `last_compiled` 過舊（超過閾值的頁面）
4. 無來源的斷言（事實性陳述缺引用）
5. 長停 stub 頁（有標記 stub 但超過 N 天未更新）
6. dead `[[wiki-link]]`（指向不存在頁面的連結）
7. `wiki/_e*` 空目錄或空頁

**理由：**
- Session 1 是完整 28 訊息的獨立工作流，不是 consume 的附帶動作
- `1-llm-wiki-knowledge-base` 提到 lint 為四個動詞之一，但沒有具體執行指引
- `2-jk-nb-consume` 已成功包成 skill；lint 是自然的下一個
- 工作流有非顯然的細節（7 種問題類型、雙輸出檔案、agent-actions append 而非覆寫）
