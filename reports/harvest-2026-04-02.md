### ★★★ Strong

#### `pitch-deck`
- **目的**：製作結構化的商業提案/投資人簡報，涵蓋封面、問題、解決方案、市場、商業模式、團隊、里程碑等標準章節
- **觸發情境**：「幫我做 BP」、「寫商業計畫簡報」、「投資人 deck」、「公司介紹簡報」
- **分類**：`docs`
- **理由**：Sessions 1（朋昶 BP）和 3（sabre 提案）都是從零開始製作業務簡報，且都走了 **確認公司背景 → 規劃章節結構 → 製作 HTML slides → build PPTX** 的完整流程。辦公室簡報（office-pptx）只管格式，而 pitch-deck 還需理解商業敘事邏輯、章節順序、各頁 copy 撰寫，是不同層次的能力。兩個 session 的結構高度相似（封面 + 議程 + 背景 + 問題 + 解法 + 市場 + 里程碑），適合抽成可複用的骨架。

### ★★ Moderate

#### `iot-factory-report`
- **目的**：接收工廠設備 IoT CSV 時序資料，分析設備運作模式（換水/清洗周期、耗能規律），辨識廠務架構，產出 HTML 視覺化圖表與 PPTX 分析報告
- **觸發情境**：「分析這份工廠資料」、「幫我看 sensor log」、「製程時序分析」、「IoT 設備資料報告」
- **分類**：`workflow`
- **理由**：Session 4 完整走過 **CSV ingestion → 時序分組分析 → 架構推論 → HTML 圖表 → PPTX 報告**，步驟固定且重複性高。UPW/RO 純水設備、壓縮機、冷凍機等廠務設備都有共同的分析模式（周期偵測、異常點標記）。目前 `office-pptx` 不含分析邏輯，`doc-to-structured-data` 主要處理非結構化文件，皆無法覆蓋此場景。

#### `gantt-slide`
- **目的**：根據里程碑清單或專案時間表，在簡報中產出 Gantt 圖 slide（HTML canvas 或 SVG，可嵌入 PPTX）
- **觸發情境**：「加一張甘特圖」、「里程碑時程表 slide」、「project timeline」、「四月底前的計畫」
- **分類**：`docs`
- **理由**：Session 3（sabre）明確需要甘特圖 slide，且甘特圖的 HTML 實作（日期範圍計算、列佈局、顏色標記）是每次重複的繁瑣工作。目前 `office-pptx` 無法自動從里程碑列表產生甘特圖；`writing-plans` 只輸出 Markdown，不生成視覺化 slide。
