## Skill 候選清單

### 🟢 Strong：`docker-compose-setup`

| 欄位 | 內容 |
|------|------|
| **名稱** | `docker-compose-setup` |
| **分類** | `backend` |
| **觸發條件** | 使用者說「用 docker 包裝」、「一鍵啟動」、「containerize」、「幫我加 Docker」 |
| **用途** | 為多服務專案（frontend + backend + optional DB）生成完整 Docker Compose 配置 |

**理由：**  
Sessions 1、4、6 三個不同專案都重複執行相同流程：
1. 分析各服務堆疊（Next.js/FastAPI/Postgres）
2. 生成各服務 `Dockerfile`（含 dev/prod 分離）
3. 生成 `docker-compose.yml`（service + volume + network）
4. 生成 `.env.example` 與 `.dockerignore`
5. 驗證 port 映射與服務間連線

現有的 `deploy` skill 專注在雲端部署平台選擇，`ci-pipeline` 專注在 GitHub Actions，兩者都**不涵蓋**本地多服務 Compose 一鍵啟動這個完整工作流程。
