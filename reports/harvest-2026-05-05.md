## Skill 候選

### 🟢 Strong：`company-name-extractor`（暫名）

- **目的**：從圖片或純文字中抽取公司名稱，輸出標準 JSON array `[{"name": "完整公司全名", "english": "英文名或null"}]`，含台灣中文公司、外商英文公司、子公司、品牌實體；排除個人姓名、產品名、技術名詞、地名。
- **Trigger**：使用者說「抓公司名」、「company extraction」、「OCR 公司」、「從圖片/文字抽公司」、「綻放計畫 OCR」、或上傳圖片並要求結構化公司清單。
- **Category**：`backend/`（接近 `doc-to-structured-data`）或 `workflow/`
- **Rationale**：
  - 8 個 session 都在做同一件事（其中 5 個在反覆微調 prompt 文案），這就是典型「該變成 skill」的訊號 — 每次重新發明 prompt 既浪費 token 又會漂移。
  - 已驗證的 prompt 規格清楚：包含/排除規則、輸出 schema、零額外文字。
  - 屬於綻放計畫的核心 backend 流程之一，後續還會持續用。
  - 與既有 `doc-to-structured-data` 技能的「結構化解析文件」哲學一致，但專攻「公司實體抽取」這個領域子題。
- **建議內容**：定版 system prompt、輸入分流（圖片走 vision、純文字走 text）、JSON 驗證（schema 校驗 + 去重 + 中英文 alias 對齊）、降級策略（OCR 模糊時回傳 partial）。
