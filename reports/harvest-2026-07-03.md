## 二、Skill 候選

### 🟢 Strong

#### 1. `persona-quiz-generator`（persona 化出題器）
- **用途**：以指定 persona（如創投視角、初學者）+ 主題技術痛點背景，生成繁中選擇題，混合題型（機制題/取捨題/RD 情境題），輸出嚴格 JSON 陣列供程式消費
- **觸發**：「幫我出 N 題選擇題」、headless 出題呼叫、學習 app 需要題庫
- **分類**：workflow
- **理由**：5 個 sessions 重複使用幾乎相同的 prompt 結構（persona 定義、難度分級、生活比喻約束、JSON schema）。這些 prompt 目前散落在呼叫端，codify 成 skill 可統一題型規範、解析深度標準與 JSON 驗證，避免每次呼叫重寫模板。現有 skills 無任何出題/測驗生成類別（qa-* 系列皆為軟體測試）

#### 2. `doc-to-learning-app`（學習計畫 → 學習追蹤 App）
- **用途**：讀取結構化學習計畫文件（本地或 Google Drive），生成含月度內容、進度追蹤、測驗整合的 web app；串接 persona-quiz-generator 作為題庫後端
- **觸發**：「基於這個學習計劃幫我做進度追蹤 app」、「把讀書計畫變成可互動的網站」
- **分類**：workflow
- **理由**：Session 9 耗費 193 則訊息、Edit 48 + Write 46 次，是本批最大工作量。流程可複製：文件解析 → task-brief 釐清需求 → mockup → React 元件（卡片、詞彙表、進度、測驗）→ headless 出題管線。gdrive-to-skills 只做「文件 → 知識 skill」，不涵蓋 app 生成

### 🟡 Moderate

#### 3. `skill-doctor`（Skill 健康檢查器）
- **用途**：安裝 skill/agent 後自動驗證：SKILL.md frontmatter 合法、hooks 有掛上、cron 有註冊、所需權限已設定、可被 Skill tool 發現；輸出 PASS/FAIL 報告
- **觸發**：「確認 X skill 是否有順利運行」、安裝新 skill 後、「為什麼這個 skill 沒觸發」
- **分類**：meta
- **理由**：Session 10 用了 34 次 Bash 手動檢查，且 session 8 安裝後也需人工測試。現有 settings-audit / audit-fix 只管權限設定，env-doctor 管專案環境，皆不涵蓋「skill 本身是否可運作」。與現有 sk-agent-doctor 檔案可能部分重疊，建議先確認該檔內容再決定是否新建
