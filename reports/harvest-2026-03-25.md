## Skill 候選分析

### Strong — 候選人分析 (candidate-analysis)

| 欄位 | 內容 |
|------|------|
| **名稱** | `candidate-analysis` |
| **用途** | 面試候選人管理：從 PDF 履歷提取結構化資料、分析 GitHub repo 程式碼品質、建立候選人 profile markdown |
| **觸發** | 使用者說「分析履歷」「建立候選人」「evaluate candidate」「看一下這個 GitHub」（面試情境） |
| **分類** | workflow |
| **依據** | Session 2 展示完整的多步驟工作流：(1) 初始化專案結構 (2) 解析 PDF 履歷 (3) 瀏覽 GitHub repo 評估程式能力 (4) 產出結構化 profile.md。此流程可重複用於每位候選人，且涉及 PDF 讀取、web-scraper、程式碼評估等跨工具整合，值得封裝。 |

**工作流步驟**:
1. 讀取 PDF 履歷 → 提取基本資訊、學歷、經歷
2. 掃描 GitHub profile → clone/瀏覽重點 repo，評估程式碼品質
3. 產出 `profile.md`（YAML frontmatter + 結構化評估）
4. 可選：與職缺 JD 交叉比對適配度

**與既有 skill 的差異**:
- `customer-intel`：對象是公司，非個人；產出是銷售情報，非面試評估
- `office-pdf`：僅 PDF 操作，無 GitHub 分析或 profile 生成邏輯
