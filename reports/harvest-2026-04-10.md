## Skill 候選評估

### Strong — 高重複、可直接編碼

#### 1. `gdoc-report-builder` — Google Docs/Slides 結構化報告產生器

- **用途**: 把結構化資料 (表格、metadata、統計) 透過 Google Drive MCP 工具批量寫入 Google Docs 或 Slides，包含精確的段落樣式、表格編輯、批次取代
- **觸發**: 使用者要求「寫進 Google Doc」、「更新簡報」、「把資料填進 Slides」
- **分類**: `docs/`
- **依據**: Session 3 (290 msgs) 和 Session 6 (638 msgs) 都大量使用 `applyParagraphStyle` (117次)、`editTableCell` (109次)、`findAndReplaceInDoc` (58次)。目前的 `office-pptx` / `office-docx` 是本地 .pptx/.docx 操作，**沒有任何 skill 封裝 Google Drive MCP 的結構化文件產出流程**。這是目前工具使用量最大的未覆蓋區域。
- **核心流程**: 讀取資料 → 建立/開啟 Google Doc/Slides → 批量 editTableCell 填表 → applyParagraphStyle 統一格式 → findAndReplaceInDoc 替換變數 → 匯出/分享

#### 2. `jd-writer` — 職缺描述 (JD) 撰寫與精修

- **用途**: 根據公司組織架構、職務定位、團隊需求，產出結構化 JD (工作內容、必備技能、加分條件、職涯發展路徑)
- **觸發**: 使用者說「寫 JD」、「補 JD」、「檢查 JD」、「職缺描述」
- **分類**: `workflow/`
- **依據**: Session 19 展示了完整的 JD 撰寫流程 — Senior SWE 和 Data Scientist 兩個職缺，需要理解公司組織 (DA → Senior DA/PM → DS or Presales) 才能寫出準確的 JD。目前 `candidate-analysis` 只處理候選人端，**缺少職缺端的結構化工具**。JD 品質直接影響招募效率，且格式高度可模板化。
- **核心流程**: 輸入職務定位 + 組織脈絡 → 產出必填欄位 (工作內容/必備技能/加分條件) → 選填欄位 (薪資範圍/職涯路徑) → 交叉比對現有 JD 一致性

#### 3. `agent-persona` — 自動化 Agent 角色模板管理

- **用途**: 為 headless agent 產生結構化的角色 prompt (tester、maintainer、reviewer 等)，包含專案結構摘要、工具權限、輸出格式
- **觸發**: 使用者設定新 agent 時、「建立 tester agent」、「新增 maintainer」
- **分類**: `meta/`
- **依據**: Sessions 8-15 (8 個 sessions, 312 msgs) 反覆使用幾乎相同的 tester/maintainer agent prompt 模板。每次都手動注入專案結構和角色定義。目前 `headless-agent` 處理排程和執行機制，但**不處理角色 prompt 的結構化產生與版本管理**。將角色模板獨立為 skill 可以：(1) 標準化 prompt 格式 (2) 自動注入最新專案結構 (3) 支援角色組合 (tester + maintainer 一起跑)。
- **核心流程**: 選擇角色模板 → 自動讀取專案結構 → 注入到模板 → 產出完整 agent prompt → 可選：直接啟動 headless agent
