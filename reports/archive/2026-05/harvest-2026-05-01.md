## 技能候選評估

### ⭐⭐⭐ 強 (Strong) — `tw-pe-due-diligence`

**名稱：** `tw-pe-due-diligence`
**用途：** 以台灣資深私募股權（PE）投資人角色，對台灣非上市公司執行初步盡職調查（Due Diligence），產出結構化中文報告。
**觸發條件：**
- 使用者提供公司名稱 + 統一編號，要求盡調分析
- 關鍵詞：「盡職調查」/ 「初步盡調」/ 「PE分析」/ 「請扮演PE投資人」
- 輸入格式包含「代表人」「實收資本額」「所屬產業」等 PE 盡調欄位
**分類：** `workflow` → 台灣財務盡調
**理由：**

此模式主導了 **30 個工作階段中的 25 個（83%）**，是所有工作階段中最高度重複的工作流。每個工作階段均遵循相同步驟序列：

```
1. tw-company-lookup  →  取得政府官方登記資料（統編、董監事、歷史變更）
2. WebSearch × 4-8   →  新聞、爭議、融資紀錄、產業分析
3. WebFetch × 2-5    →  深讀特定報導或公開資料
4. 撰寫結構化報告      →  業務模式、財務概況、法律風險、PE評估
```

**現有技能缺口分析：**

| 現有技能 | 差異說明 |
|----------|---------|
| `tw-company-lookup` | 僅查詢政府登記資料，不做 PE 分析或市場研究 |
| `investment-research` | 針對上市股票+技術分析+回測，非私募非上市公司 |
| `customer-intel` | B2B 銷售情蒐，同步 Nexus API，聚焦會前準備而非投資評估 |

PE DD 報告需要的獨特能力：
- 以投資人視角評估「護城河」、「退出路徑」、「估值合理性」
- 辨識融資輪次、法人股東結構、關係人交易風險
- 台灣特有的法規查核（公司法違規、行政裁罰）
- 最終需輸出「投資評級」與「主要風險標記」

### ⭐⭐ 中 (Moderate) — `tw-company-entity-extractor`

**名稱：** `tw-company-entity-extractor`
**用途：** 從非結構化文字或圖片（截圖、PDF 頁面）中識別並擷取台灣公司/機構名稱與統一編號，回傳標準化 JSON 陣列。
**觸發條件：**
- 使用者貼上一段文字，請求找出其中的公司名稱
- 使用者提供圖片路徑，要求辨識其中的公司資料
- 關鍵詞：「找出公司名稱」/ 「回傳 JSON」/ 「辨識股份有限公司」
**分類：** `backend` → 台灣資料處理
**理由：**

此模式出現在 **4 個工作階段**（工作階段 7、14 為純文字；16、21 為圖片）。兩種子模式皆具明確且可複用的規則：

```
文字輸入路徑：
  → 辨識「股份有限公司」/「有限公司」標準結尾
  → 辨識看似組織但無標準結尾的名稱
  → 回傳 JSON 陣列，不捏造

圖片輸入路徑（Read tool multimodal）：
  → 若有「統一編號」欄位 → {"公司名稱": ..., "統一編號": ...}
  → 若無統一編號 → 只回傳名稱陣列
```

雖然邏輯相對簡單，但此模式是工作階段 12 自動化盡調系統的**核心前處理步驟**，且在人工使用中也反覆出現。若能標準化提示與輸出格式，可減少每次重新定義規則的成本。

**保留疑慮：** 邏輯足夠簡單，存在作為獨立技能是否有必要的疑問。若 `tw-pe-due-diligence` 技能已涵蓋此功能為一個子步驟，可考慮不另立技能。
