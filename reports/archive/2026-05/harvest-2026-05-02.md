## 二、Skill 候選人分析

### 🟢 Strong — `claude-cli-python-client`

**用途：** 在 Python 後端中透過 subprocess 呼叫 Claude Code CLI，支援純文字及圖片推論。自動探索 CLI 路徑（Windows / WSL / bun / npm 多路徑掃描）。

**觸發條件：** 使用者想在 Python 應用程式中使用 Claude AI，但不使用 API key（改用 Claude Desktop 認證）；或問「如何從 Python 呼叫 Claude CLI」。

**分類：** `backend`

**理由：** `claude_client.py` 在整個 taiwan-company 專案中是核心模組，被 `company_extractor`、`memo_extractor` 等多個服務引用。其路徑探索邏輯（涵蓋 bun cache、npm global、WSL 路徑）是非顯而易見的工程知識，且與現有 `claude-api` skill（使用 API SDK）完全互補——這套方案零 API key、走 CLI。**不重複任何現有 skill。**

---

### 🟢 Strong — `investment-call-memo`

**用途：** 從面談記錄文件（PDF / DOCX / 圖片）中，提取結構化的投資案面談備忘錄欄位：受訪者、資本額、IPO 時程、財務狀況、主要客戶、競爭者、結論等 25+ 欄位。

**觸發條件：** 使用者說「幫我整理這份面談紀錄」、「從這個 PDF 提取 call memo」、「我要建立投資備忘錄」。

**分類：** `workflow`

**理由：** `call_memo.py` 定義了高度結構化的 `MemoSave` model（25 個財務/盡調欄位），`memo_extractor.py` 透過 Claude 將非結構化文字轉換為這些欄位。這是 VC/PE 特有的工作流程，現有 `investment-research`（查公開資料）和 `candidate-analysis`（HR）都不涵蓋此場景。**不重複任何現有 skill。**

---

### 🟢 Strong — `multi-format-chinese-doc-parser`

**用途：** 從 PDF、DOCX、Excel、圖片中提取繁體中文文字。圖片優先使用 Windows PowerShell 內建 OCR（對中文支援最佳），fallback 到 Tesseract；同時處理 WSL 路徑映射。

**觸發條件：** 使用者需要解析含中文的文件檔案；問「如何在 WSL/Windows 做中文 OCR」；或需要多格式文件解析。

**分類：** `backend`

**理由：** `file_parser.py` 涵蓋了 fitz (PDF)、python-docx、openpyxl、Windows MFC OCR 的完整中文文件解析套路，且正確處理 WSL ↔ Windows 路徑問題。`doc-to-structured-data` 已存在但不包含中文 OCR 的具體實作細節。**補充現有 skill 的盲點。**

---

### 🟡 Moderate — `entity-classifier-from-doc`

**用途：** 上傳任意格式文件 → 提取指定類型的實體（公司名稱、人名、產品等）→ 用 Claude AI 按規則分類為 valid / excluded / uncertain 三組。

**觸發條件：** 使用者需要「從一份文件裡找出所有 X 並分類」；或需要建立文件 → 實體分類的 FastAPI 端點。

**分類：** `backend`

**理由：** `company_extractor.py` 的三組分類邏輯（valid/excluded/uncertain）加上去重、資料庫比對是一個可複用的模式，但其核心模式（文件 → AI → 結構化）與 `doc-to-structured-data` 有一定重疊。差異在於：本模式含分類決策樹和不確定項目的人工確認流程。**輕度重疊，仍有獨特價值。**

---

### 🟡 Moderate — `batch-industry-tagger`

**用途：** 給定公司清單（ID：公司名稱 — 業務簡介），用 Claude AI 分批判斷哪些符合特定產業別（如循環經濟、綠能、AI等），輸出符合條件的 ID JSON 陣列。

**觸發條件：** 使用者有一份公司清單，要做產業分類、打標籤、或篩選特定產業。

**分類：** `workflow`

**理由：** Session [2] 展示了一個乾淨的 single-prompt 模式：格式化公司清單 → Claude 輸出 JSON → 無多餘說明文字。這個模式可泛化到任何「從清單中篩選符合條件的項目」場景，但目前尚未成熟到需要完整 skill（只有 1 個 session 展示此模式）。**待更多 session 驗證後再建立。**
