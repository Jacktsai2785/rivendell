- **分類**：workflow

```

  → WebFetch（詳細內容、官網、媒體報導）
  → 結構化輸出：公司概覽、產品服務、市場地位、財務狀況、風險分析、投資建議
```

#### 2. `systemd-user-service`

- **目的**：為 Linux 環境下的 Web App（FastAPI + 前端框架）設定 systemd user service，實現開機自啟、崩潰自動重啟
- **觸發時機**：使用者說「設計成像 taiwan-company 一樣用 systemd 管理」或「設定服務開機自動啟動」或排查 systemd 服務未啟動問題
- **分類**：backend
- **評分**：Strong ★★★

**佐證 Session：** [19]（建構）、[23]（除錯）

**工作流程：**
```
偵測專案結構（FastAPI / Next.js / Streamlit ...）
  → 產生 ~/.config/systemd/user/*.service 檔案
  → systemctl --user daemon-reload && enable --now
  → 驗證服務狀態 + journalctl 檢查日誌
  → 設定 loginctl enable-linger（非 root 自啟）
```

**理由：**
Session [19] 和 [23] 都花費大量時間設定與除錯 systemd 服務，且 Session [19] 明確說「請與 taiwan-company 專案一樣，設計成 systemd user service」，顯示這是跨專案重複出現的需求。現有 `launchd-agent` 只涵蓋 macOS，Linux 環境缺乏對應的 skill。
