## Skill 候選分析

### 1. `doc-to-structured-data` — Moderate

| 欄位 | 內容 |
|------|------|
| **名稱** | doc-to-structured-data |
| **用途** | 將非結構化技術文件（.doc/.pdf 測試計畫、規格書、datasheet）解析為結構化 CSV/JSON |
| **觸發** | 使用者要求「結構化」、「轉 CSV」、「解析文件」、「提取規格」，或提供 .doc/.pdf 技術文件要求整理 |
| **類別** | workflow 或 docs |
| **強度** | **Moderate** |

**理由**：
- richwave session 展現了完整的多步驟工作流：格式探測 → 萃取策略選擇 → 領域結構辨識 → 分表輸出 → 驗證
- 涉及的領域知識包含：IC 測試計畫結構（test spec、bin definition、test flow）、.doc 格式處理策略（textutil / olefile / PDF fallback）
- 此模式可泛化至其他技術文件類型（datasheet、規格書、BOM 表）

**降級原因**：
- 僅出現 1 次，尚未確認重複性
- 領域結構（IC 測試計畫的欄位定義）高度依賴文件類型，難以寫出通用 skill
- 現有 `office-docx` 和 `office-pdf` 已涵蓋部分文件讀取能力

**建議**：觀察後續是否再出現類似的「文件 → 結構化資料」需求。若 richwave 專案持續有此類任務，可考慮建立專案級 skill（如 `ic-testplan-parser`）而非通用 skill。
