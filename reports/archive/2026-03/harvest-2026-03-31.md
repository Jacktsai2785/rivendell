## Skill 候選清單

### ⭐⭐⭐ Strong（強力候選）

#### 1. `audio-transcription`

| 欄位 | 內容 |
|------|------|
| **目的** | 在 Web App 中加入音訊上傳 → 語音辨識 → 逐字稿顯示的完整流程 |
| **觸發時機** | 使用者說「我要上傳音檔並轉成逐字稿」、「語音辨識」、「Whisper 整合」 |
| **分類** | backend / workflow |
| **依據** | Session 14：建立了 `audio-transcription-flow.md`、`audio-transcription.html`、`audio-transcription.md`，顯示完整的 UI + API + 轉換流程設計，且是跨前後端的固定模式 |
| **與現有 skills 差異** | 現有 `imap-smtp-integration` 類似「協議整合」模式，但無音訊處理 skill |

**建議 skill 內容：**
- 技術棧選擇（OpenAI Whisper API / local whisper / Google Speech-to-Text）
- 前端：file input → FormData upload → 進度顯示
- 後端：multipart 接收 → 轉換 → 儲存逐字稿
- 常見格式（mp3, m4a, wav）與大小限制處理

#### 2. `github-repo-audit`

| 欄位 | 內容 |
|------|------|
| **目的** | 審計本地 projects 目錄與 GitHub remote 的一致性，找出 obsolete / 未 push / 需刪除的 repo |
| **觸發時機** | 「GitHub repo 還有必要嗎」、「哪些 repo 已經沒用」、「local 跟 remote 不一致」 |
| **分類** | git / meta |
| **依據** | Session 12：完整走過 gh repo list → 比對本地目錄 → 判斷 active/obsolete → 清理 → 同步 push 的流程（56 則訊息的完整 workflow） |
| **與現有 skills 差異** | `repo-rename` 處理改名；`auto-stage` 處理 staging；均未涵蓋跨 repo 健康審計 |

**建議 skill 內容：**
- `gh repo list` 取得 remote 清單
- 比對本地 `~/Documents/Projects/` 目錄
- 分類：active（有近期 commit）/ stale（久無更新）/ orphan（local 有但 remote 無，或反之）
- 建議行動：push、archive、delete
