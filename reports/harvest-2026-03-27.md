## 技能候選人分析

### 1. `family-ledger` — 家庭財務借貸管理

- **評級**: ⚡ Strong
- **用途**: 建立簡易家庭借貸/贈與記帳系統 — SQLite DB、CLI 操作、自動生成借貸合約 PDF/Markdown
- **觸發**: 使用者說「家庭記帳」「借貸管理」「誰欠誰錢」「生成借據」或 `/family-ledger`
- **分類**: `workflow`
- **理由**:
  - Session 1 展示完整的端到端工作流：需求釐清 (AskUserQuestion x3) → DB schema 設計 → CLI 實作 → 合約生成
  - 涉及特定領域知識：借貸合約格式、利息計算邏輯、贈與 vs 借貸的區分
  - 可重用於任何小型財務追蹤場景
  - **現有技能無覆蓋**

### 2. `python-cli-scaffold` — Python CLI 應用腳手架

- **評級**: 🔶 Moderate
- **用途**: 快速搭建 Python CLI 應用 — argparse/click、SQLite DB layer、模組結構
- **觸發**: 使用者說「建立 CLI 工具」「Python 命令列」「CLI scaffold」或 `/python-cli-scaffold`
- **分類**: `backend`
- **理由**:
  - Session 1 中建立了 main.py + db.py + contract.py 的典型三層結構
  - 此模式在多個專案中反覆出現（scraper、projection 等都是 Python CLI）
  - 但範圍可能太廣泛，且 `init-project` skill 已有部分覆蓋
  - **與 init-project 有部分重疊，需要差異化**

### 3. `skill-categorization` — Skills 批量分類與 Dashboard 同步

- **評級**: 🔶 Moderate
- **用途**: 分析未分類 skills，根據功能自動建議分類，批量搬移並更新 dashboard 資料
- **觸發**: 使用者說「歸類 skills」「未分類太多」「整理技能庫」或 `/skill-categorization`
- **分類**: `meta`
- **理由**:
  - Session 4 展示了 22 個 skills 的分類流程：讀取 TSV → 分析功能 → 決定分類 → 批量 mv → 更新 page.tsx
  - 此操作隨技能庫增長會越來越頻繁
  - 但可能只是 `skill-creator` 的延伸功能，不值得獨立 skill
  - **頻率不確定，可併入 skill-creator**

### 4. `crm-sync-verify` — CRM 投影驗證與修復

- **評級**: 🔵 Weak
- **用途**: 驗證 crm-projection 產出的客戶 markdown 是否完整、frontmatter 格式正確、deal 資料同步
- **觸發**: crm-projection 執行後自動觸發，或使用者說「檢查客戶資料」
- **分類**: `quality`
- **理由**:
