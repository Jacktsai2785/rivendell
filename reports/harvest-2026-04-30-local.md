## Skill 候選清單

### 1. `pe-dd-taiwan` — 強 (Strong)

**目的：** 以台灣私募股權（PE）投資人視角，對指定台灣公司進行初步盡職調查，輸出結構化 DD 報告。

**觸發條件：**
- 使用者提供：公司名稱 + 統一編號 + 所屬產業 + 代表人
- 提示包含：「盡職調查」、「Due Diligence」、「PE投資人」、「初步DD」

**分類：** `workflow`

**工作流程（從 sessions 歸納）：**
1. 以公司名稱 + 統一編號進行 WebSearch，取得基礎資訊
2. 搜尋代表人背景、過往職歷
3. 搜尋公司近期新聞、訴訟、政府獎補助
4. 搜尋競爭對手、產業地位
5. 搜尋主要客戶/合作夥伴
6. 調用 `/tw-company-lookup` 取得政府登記資料（可選）
7. 輸出結構化報告（優勢、風險、建議）

**為何不重複現有 skill：**
- `tw-company-lookup`：僅查政府登記資料，無法做競爭分析或風險評估
- `investment-research`：針對上市股票，使用 news_stock 資料庫，非 PE 私募視角
- `customer-intel`：以銷售情報為主，非投資 DD 框架

**重複頻率：** 6 個獨立 sessions，每次格式高度一致，包含相同的角色設定提示詞

### 2. `company-name-extractor` — 中 (Moderate)

**目的：** 從非結構化文字（PDF 擷取文字、OCR 輸出、新聞稿等）中辨識並擷取所有公司/機構名稱，輸出 JSON 陣列。

**觸發條件：**
- 「找出所有疑似公司或機構的名稱」
- 「從以下文字中找出公司名稱」
- 輸入為大段文字 + 要求輸出 JSON

**分類：** `workflow`

**工作模式（從 sessions 歸納的規則）：**
1. 包含「股份有限公司」或「有限公司」的名稱 → 完整保留
2. 看起來像公司但沒有標準結尾（如縮寫機構名）→ 列出
3. 不得捏造不存在的名稱
4. 輸出格式：純 JSON 陣列 `["A公司", "B機構"]`，無其他說明文字

**為何值得作為 skill：**
- 7 個 sessions 完全相同任務，且此任務為 Session 18 主系統的關鍵子流程
- 規則有細節（不捏造、保留完整名稱、處理非標準名稱），適合編碼為 skill 提示
- 目前 `doc-to-structured-data` 偏向欄位提取，非專門的公司實體辨識

### 3. `tw-company-profile-writer` — 中 (Moderate)

**目的：** 根據公司基本資料（統一編號、產業、代表人、資本額等），撰寫一段 300-500 字、以台灣商務中文寫作的專業公司簡介。

**觸發條件：**
- 「撰寫公司簡介」
- 提供公司基本資料 + 要求輸出中文簡介
- 指定 300-500 字、商務中文

**分類：** `docs`

**撰寫規則（從 sessions 歸納）：**
1. 涵蓋：經營業務、核心價值、市場地位、主要競爭優勢
2. 使用專業、客觀、精煉的台灣商務中文（非簡中）
3. **禁止捏造**不存在的業務或數據
4. 資訊不足時，可就已知資料做合理推論，明確標示「依現有資料推測」
5. 字數：300-500 字

**為何值得作為 skill：**
- 10 個 sessions 完全相同任務（佔總 sessions 的 1/3）
- 有明確的「禁止捏造」限制，需要 skill 提醒模型不要幻覺
- 不同於 `doc-coauthoring`（協作寫作）或 `sow-writer`（報價單）
